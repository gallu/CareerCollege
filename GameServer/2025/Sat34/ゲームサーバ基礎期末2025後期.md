## 前提

すべての「ダメージ計算」「命中判定」の問題のメソッドは、引数を「攻撃側PCインスタンス」「防御側PCインスタンス」としなさい。  
必要なパラメタは、プロパティへのアクセス(その場合、プロパティはreadonlyであることが望ましい)またはgetterメソッドで行うものとします。  
「全設問で通用する汎用のPCクラス」でも「個々の設問毎に別々のPCクラス」でもどちらでもOKとします。  
PCクラスと回答用のクラスは、１ファイルにまとめなさい。もし「全設問で通用する汎用のPCクラス」の場合、「pc.phpを作成し」「各ファイルで require_once」でもよい、とします。  

例えば、問1用のPCクラスであれば、以下のいずれかのPCクラスのような感じになります(以下以外でも、要件を満たしていればOKです)。  

```
class PC {
    private int $attack; // 攻撃力
    private int $defense; // 防御力
    public function __construct(int $attack, int $defense) {
        $this->attack = $attack;
        $this->defense = $defense;
    }
    public function getAttack(): int {
        return $this->attack;
    }
    public function getDefense(): int {
        return $this->defense;
    }
}
```

## (問1)

次の仕様に従って、ダメージを計算するクラスを作成しなさい。  
クラス名およびメソッド名はなんでもよいが、例えば `Battle` クラス、`calcDamage()` メソッド、などはわかりやすい。  

### 仕様

ダメージは、以下の数式で計算されるものとする。

```
ダメージ = ((攻撃力 ÷ 2) - (防御力 ÷ 4)) × 乱数補正
```

### 条件

- メソッドの引数は「攻撃側PCのインスタンス」「防御側PCのインスタンス」であること
- 計算したダメージは、メソッドの戻り値として返すこと
- ダメージで小数点が出た場合は切り捨てにすること

## (問4)

次の仕様に従って、攻撃が命中したかどうかを判定するクラスを作成しなさい。
クラス名およびメソッド名はなんでもよいが、例えば `Battle` クラス、`isHit()` メソッド、などはわかりやすい。  

### 仕様

命中判定は、以下の手順で行うものとする。

1. 攻撃者の命中判定を行う。命中判定に失敗したら外れ
2. 防御者の回避判定を行う。回避判定に成功したら外れ
3. 「命中判定に成功」かつ「回避判定に失敗」したら命中

メソッドの戻り値は、命中した場合は `true`、
命中しなかった(外れの)場合は `false` とする。

### 判定方法

- 命中判定・回避判定はいずれも **パーセントロール** とする
- パーセントロールは以下の方法で行う

```
1 ～ 100 の乱数を生成し、その値が引数以下であれば成功
```

### 条件

- メソッドの引数は「攻撃側PCのインスタンス」「防御側PCのインスタンス」であること
    + 攻撃者の命中率、防御者の回避率が必要なパラメタになる
- 最終的な命中結果を `bool` 型で返すこと
- 乱数は毎回新しく生成すること

## (問5)

次の仕様に従って、攻撃が命中したかどうかを判定するクラスを作成しなさい。
クラス名およびメソッド名はなんでもよいが、例えば `Battle` クラス、`isHit()` メソッド、などはわかりやすい。  

### 仕様

命中判定は、以下の手順で行うものとする。  
なお、1d20とは「20面ダイスを振る」ことで、1～20の間の乱数を作成することを意味する。  

1. 1d20 + 命中ボーナス で、「命中値」を算出する
2. 1d20 + 回避ボーナス で、「回避値」を算出する
3. 命中値 >= 回避値 なら命中、そうでなければ外れ

メソッドの戻り値は、命中した場合は `true`、
命中しなかった(外れの)場合は `false` とする。

### 条件

- メソッドの引数は「攻撃側PCのインスタンス」「防御側PCのインスタンス」であること
    + 攻撃者の命中ボーナス、防御者の回避ボーナスが必要なパラメタになる
- 最終的な命中結果を `bool` 型で返すこと
- 乱数は毎回新しく生成すること
